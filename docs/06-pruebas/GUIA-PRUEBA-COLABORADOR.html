Wings - Guia de Prueba Funcional

Documento para que un colaborador pruebe el sistema de punta a punta, registre fallas y ayude a definir seeders realistas. Esta prueba no busca que todo salga bien: busca descubrir exactamente que funciona y que no.

Preparacion Datos minimos Checklist Flujos completos Seguridad Reporte

1. Preparacion

Antes de empezar
  • Entrar a la URL indicada por Carlos o por el equipo.
  • Usar navegador Chrome o Edge actualizado.
  • No corregir datos durante la prueba sin anotarlo.
  • Tomar captura de pantalla ante cada falla.
  • Registrar el usuario/rol usado en cada caso.
Usuarios esperados
  • Admin.
  • Operativo.
  • Profesor.

Las credenciales las debe entregar quien prepara la prueba.

Importante

Si un dato necesario no existe, no inventar cualquier cosa: registrar que falta dato de prueba. Eso sirve para crear seeders utiles.

4. Flujos Completos Obligatorios

A. Alumno -> Deuda -> Cobro -> Caja -> Cashflow

  1. Entrar como operativo.
  2. Elegir alumno con deuda pendiente.
  3. Cobrar cuota.
  4. Confirmar que se abre caja si no existia.
  5. Ver resumen y detalle de caja.
  6. Cerrar caja.
  7. Entrar como admin.
  8. Validar caja.
  9. Confirmar movimiento en cashflow.

B. FIFO y pago parcial

  1. Elegir alumno con deuda vieja y deuda nueva.
  2. Intentar pagar deuda nueva sin completar vieja.
  3. Confirmar rechazo.
  4. Pagar vieja completa y dejar ultima parcial.
  5. Confirmar saldos.

C. Clase -> Asistencia -> Liquidacion -> Pago

  1. Entrar como admin y crear clase.
  2. Entrar como profesor u operativo y cargar asistencia.
  3. Entrar como admin y generar liquidacion.
  4. Revisar detalle.
  5. Cerrar y pagar liquidacion.
  6. Revisar recibo o impacto en cashflow.

D. Caja rechazada

  1. Operativo carga movimientos y cierra caja.
  2. Admin rechaza caja.
  3. Operativo edita o agrega movimiento.
  4. Operativo vuelve a cerrar.
  5. Admin valida.
